# Headway 自動化內容擷取工具：開發總結報告

這份報告詳盡記錄了從概念成型到最終完成「Headway 書籍深層抓取器」的開發全過程，旨在總結技術突破與開發經驗。

---

## 🎯 專案目標
開發一個高度自動化、具備動態識別能力的 **Headway** 書籍擷取工具。讓使用者能在瀏覽器中自由選擇書籍，並透過一鍵點擊，自動將網頁端的多頁摘要、封面圖、作者資訊完整轉換為個人本地的數位知識庫。

## 🛠️ 核心技術及框架 (🛠️ 技術棧)
- **程式語言**：Python 3.9+
- **自動化框架**：[Playwright](https://playwright.dev/python/) (Chromium) - 用於模擬真實瀏覽器行為、自動化翻頁及內容渲染攔截。
- **環境管理**：Python Virtual Environment (venv) & `python-dotenv`。
- **前端注入**：JavaScript (Vanilla JS) - 用於在 Headway 背景中動態植入觸發按鈕，並透過全域屬性 `window.__headway_ready__` 與 Python 端溝通。
- **版本控制**：Git & GitHub。

## 📅 開發過程總結 

### 1. 專案規劃與初始化
最初構思為簡單的「網頁下載按鈕」注入，但隨後發現瀏覽器端的 Blob 下載無法有效被 Playwright 攔截。因此調整為「Python 主控」模式，由 Python 負責瀏覽導航與文件存檔。

### 2. 遇到的技術挑戰與卡關處
- **Blob 下載攔截失敗**：這是前期最大的卡關處。最初嘗試在瀏覽器端生成資料，但 blob 協議無法被抓取。最終修復方案是改回 Python 端用 `page.evaluate` 取得字串，再由 Python `open()` 直接寫入檔案。
- **SPA 單頁切換導致按鈕消失**：Headway 是高度動態的 SPA 網站。最終我們使用了高頻監聽 (`framenavigated`) 與手動重新注入雙保險，來確保按鈕永遠出現在左側位置。
- **代碼意外毀損修復**：在多次功能合併中曾發生過代碼重疊導致語法失效（TargetClosedError），最終透過徹底重寫並精簡邏輯結構解決。

### 3. 用戶重要提問與應對
- **「為什麼我的檔案消失了？」**：說明絕對路徑機制並強化寫檔異常捕捉。
- **「我想下載封面圖片」**：開發出透過 `page.request` 獲取圖片內容的功能並自動合併進 Markdown。
- **「如果遇到 Conclusion 就自動停止」**：開發了關鍵字比對機制，並加入 `Page > 3` 的門檻以避免早期誤判。

### 4. 最終達成成果與主要功能
- **一鍵式捕捉**：動態偵測當前網址與書名。
- **智慧清理**：自動過濾 UI 導覽文字與時間戳。
- **資產管理**：自動建立專屬資料夾並同步下載封面 (`cover.jpg`)。
- **格式優化**：輸出的 Markdown 筆記結構統一，內建封面圖片預覽。

## 如何使用或啟動
1. 確保 `.env` 資料正確。
2. 執行 `./run.sh` 開啟瀏覽器。
3. 導航至喜歡的一本書。
4. 點選左上角桃紅色的 **[📥 開始抓取此書內容]**。
